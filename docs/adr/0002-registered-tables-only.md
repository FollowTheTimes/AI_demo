# ADR 0002: 数据源表必须使用已注册表

## 状态
已采纳

## 背景
LLM生成 `.cube` 模型时需要引用数据源表（如 `tt.jz_bank_bill`）。存在两种策略：

1. **仅用已注册表**：LLM只能引用系统中已注册的数据源表
2. **允许创造新表**：LLM可以生成新的表名，后续由人工在平台中创建

## 决策
采用 **仅用已注册表** 方案。

## 理由

1. **平台约束**：公安大数据研判分析平台只能查询已注册的数据源表，引用不存在的表会导致模型运行失败
2. **数据安全**：政务环境中不能随意引用未经验证的数据源
3. **可预测性**：限制数据源范围使生成结果更可控、更可预测
4. **可扩展**：需要新表时，先在 schema 中注册，LLM即可引用

## 后果

- 需要维护一份已注册数据源表的 schema 定义（`knowledge/schemas/bank_tables.json`）
- LLM prompt 中必须包含可用表清单及字段定义
- 新增数据源表时需更新 schema 文件
- 如果用户需求涉及未注册表，系统应提示用户先注册表
- 数据源表定义的唯一权威来源为 SchemaRegistry（`knowledge/schemas/bank_tables.json`），config.py 中的 DATA_SOURCE_TABLES 已删除（ADR-0006）
