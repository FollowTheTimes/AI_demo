# ADR 0003: 自动校验+重试策略

## 状态
已采纳

## 背景
LLM生成的 `.cube` 文件可能存在格式问题（JSON不合法、缺少必填字段、引用了不存在的数据源表等）。存在三种处理策略：

1. **自动校验+重试**：检查不合法则自动重新生成
2. **校验+报错**：检查不合法则返回错误信息让用户重新描述
3. **不校验**：直接输出LLM生成的结果

## 决策
采用 **自动校验+重试** 方案，最多重试3次。

## 理由

1. **用户体验**：一步生成的要求意味着用户不应看到中间错误，系统应自动修复
2. **LLM不确定性**：大模型输出有随机性，重试往往能得到合法结果
3. **成本可控**：限制重试次数（3次）避免无限循环
4. **降级策略**：3次重试仍失败则返回最佳尝试结果+错误提示，用户可手动修正

## 后果

- 需要实现完整的 `.cube` 文件校验器（JSON格式、必填字段、数据源表名、字段名合法性）
- 生成延迟可能因重试而增加
- 需要在 prompt 中加入更严格的格式约束以降低重试率
- 校验器使用 parse_script（ADR-0006）统一处理 script 字段的 str/dict 双态，避免解析遗漏
- 校验前使用 normalize_tables（ADR-0007）将 tables 统一转为字典格式，消除格式兼容代码
