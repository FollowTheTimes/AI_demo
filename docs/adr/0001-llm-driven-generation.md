# ADR 0001: LLM驱动生成 vs 模板填充

## 状态
已采纳

## 背景
系统需要根据用户自然语言描述自动生成 `.cube` 模型文件。现有12个手工创建的模板文件可作为参考。存在两种生成策略：

1. **模板填充**：深拷贝现有模板，修改筛选条件和字段
2. **LLM驱动生成**：让LLM直接生成完整的 `.cube` JSON结构，模板仅作为 few-shot 示例

## 决策
采用 **LLM驱动生成** 方案。

## 理由

1. **需求要求**：用户明确要求能生成现有12种类型之外的全新模型，模板填充无法满足
2. **灵活性**：LLM可以理解用户意图并创造性地组合数据源表、筛选条件、扩展列，不受模板限制
3. **可扩展性**：新增数据源表时无需创建新模板，只需更新 schema 定义
4. **模板仍保留价值**：12个模板作为 few-shot 示例，让LLM学习 `.cube` 文件的结构规律，显著提升生成质量

## 后果

- 需要精心设计 LLM prompt，包含模板示例和数据源 schema
- 需要校验层确保生成结果格式合法
- 对内网大模型的能力有一定要求（需支持 JSON 格式输出）
- 生成延迟可能高于模板填充（需等待LLM推理）
