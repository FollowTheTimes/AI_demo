# ADR 0004: 微调仅限筛选条件

## 状态
已采纳

## 背景
用户生成模型后可能需要微调。存在三种微调范围：

1. **只能调筛选条件**：用户可修改 where 条件，但不能改表结构
2. **可以改所有内容**：用户可修改增删表、修改关联关系等
3. **调条件+扩展列，改表重新生成**：部分可调，结构性变更需重新生成

## 决策
采用 **只能调筛选条件** 方案。

## 理由

1. **简单可控**：筛选条件是结构化数据（字段+操作符+值），容易提供UI编辑
2. **避免破坏**：修改表结构、克隆关系等可能导致模型无法运行
3. **符合工作流**：结构性变更应重新描述需求让AI生成，而非手动修改
4. **平台兼容**：现有平台的"右键修改条件"功能已支持筛选条件微调

## 后果

- 微调UI只需实现条件编辑器（字段选择、操作符、值输入）
- 如果用户需要结构性变更，需重新调用生成接口
- 生成的模型应尽量准确，减少用户微调需求
