# ADR 0005: 先手动导入后期自动化

## 状态
已采纳

## 背景
生成的 `.cube` 文件需要导入公安大数据研判分析平台。存在两种策略：

1. **自动导入**：生成后自动调用平台API导入
2. **先手动，后期自动化**：先生成文件让用户手动导入，后期确认API后再自动化

## 决策
采用 **先手动，后期自动化** 方案。

## 理由

1. **API不确定**：平台导入API的具体地址、参数格式尚未确认，需要通过F12抓包获取
2. **降低风险**：手动导入阶段可以先验证生成文件的质量，确认可用后再自动化
3. **渐进式交付**：先跑通核心流程（生成），再扩展辅助流程（导入）
4. **平台兼容**：平台已有"导入模型"按钮，手动导入零成本

## 后果

- 第一版不包含自动导入功能，`datacube_importer.py` 保留接口但暂不启用
- 需要用户通过F12抓包确认平台导入API的详细信息
- 后期只需配置API地址和参数即可启用自动导入
